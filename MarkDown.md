# Основные конструкции Markdown

Markdown — это облегчённый язык разметки, созданный для максимально простого и читаемого написания текстов, которые затем могут быть преобразованы в HTML. Он идеально подходит для документации, README-файлов, заметок и описания требований. Все конструкции интуитивно понятны и легко запоминаются.

---

## Заголовки

Заголовки создаются с помощью символа решётки `#`. Количество решёток соответствует уровню заголовка (от 1 до 6). После решёток обязательно ставится пробел.

```markdown
# Заголовок первого уровня
## Заголовок второго уровня
### Заголовок третьего уровня
#### Заголовок четвёртого уровня
##### Заголовок пятого уровня
###### Заголовок шестого уровня
```

## Выделение текста

Курсив обрамляется одним нижним подчёркиванием _ вокруг текста:

```markdown
Это *курсив*
``` 

Это *курсив*

Полужирный — двумя звёздочками **:

```markdown
Это **жирный**
```

Это **жирный**

Полужирный курсив — нижним подчеркиванием и двумя звездами или тремя звездами:

```markdown
А это _**все вместе**_
```

А это _**все вместе**_

Зачёркнутый текст — двумя тильдами ~~:

```markdown
~~Зачеркнутый текст~~
```

~~Зачеркнутый текст~~ (это расширение, работает не везде, но на GitHub поддерживается).

## Списки

### Маркированный список

Используйте символы * или - в начале строки. После символа ставится пробел.

Для вложенных списков сделайте сделайте отступ в 2 пробела

```markdown
* Пункт первый
* Пункт второй
    * Подпункт 2.1
    * Подпункт 2.2
        * Подпункт 2.2.1
+ Через плюс
- Через минус
```
* Пункт первый
* Пункт второй
    * Подпункт 2.1
    * Подпункт 2.2
        * Подпункт 2.2.1

+ Через плюс
- Через минус

### Нумерованный список

Используйте цифры с точкой. Нумерация автоматическая, можно везде ставить 1.

```markdown
1. Первый пункт
2. Второй пункт
    1. Подпункт 2.1
    1. Подпункт 2.2
        1. Подпункт 2.2.1
1. Третий пункт
```

1. Первый пункт
2. Второй пункт
   1. Подпункт 2.1
   1. Подпункт 2.2
      1. Подпункт 2.2.1
1. Третий пункт

## Ссылки

### Автоматическая ссылка

```markdown
<https://systems.education/>
```

Пример: <https://systems.education/>

### Прямая ссылка с подсказкой

```markdown
[Школа Бескова и Богачёвой](https://systems.education/) без подсказки

[Школа Бескова и Богачёвой](https://systems.education/ "Переход на сайт школы") с подсказкой
```

Примеры:

* [Школа Бескова и Богачёвой](https://systems.education/) без подсказки
* [Школа Бескова и Богачёвой](https://systems.education/ "Переход на сайт школы") с подсказкой

### Справочный способ (со сносками)

Удобно, если одна ссылка используется в документе много раз

```markdown
[Школа Бескова и Богачёвой][1]
[Каталог курсов][2]

[1]: https://systems.education/
[2]: https://systems.education/b2c-ekatalog#next
```

* [Школа Бескова и Богачёвой][1]
* [Каталог курсов][2]

[1]: https://systems.education/
[2]: https://systems.education/b2c-ekatalog#next

## Таблицы

Таблицы создаются с помощью вертикальных черт | и дефисов для отделения заголовка. Двоеточия : в строке с дефисами задают выравнивание столбцов (по левому краю, по центру, по правому).

```markdown
| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|:------------|:-----------:|------------:|
| по левому   | по центру   | по правому  |
| ячейка      | ячейка      | ячейка      |
```

| Заголовок 1 | Заголовок 2 | Заголовок 3 |
|:------------|:-----------:|------------:|
| по левому   | по центру   | по правому  |
| ячейка      | ячейка      | ячейка      |

## Цитаты

Для выделения цитат используется символ > в начале строки. Можно создавать вложенные цитаты, добавляя ещё один >.

``` markdown
> Цитата
>> Вложенная цитата для уточнения
```
> Взял нож - режь, взял дошик - ешь
>> Джейсон Стэтхем

## Горизонтальная линия

Горизонтальная линия (разделитель) создаётся тремя или более дефисами ---, звёздочками *** или нижними подчёркиваниями ___ на отдельной строке.

``` markdown
Текст выше.

---

Текст ниже.
```

Текст выше.

---

Текст ниже.

## Экранирование
Если нужно вывести служебный символ как обычный текст (например, звёздочку), перед ним ставится обратная косая черта \:

``` markdown
\*звёздочка\* не будет форматированием.
```

\*звёздочка\* не будет форматированием.
