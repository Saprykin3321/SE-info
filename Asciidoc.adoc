= Основные конструкции AsciiDoc
:author: Команда аналитиков
:revnumber: 1.0

AsciiDoc: Это язык разметки, предназначенный для создания технической документации, книг, статей и спецификаций. В отличие от Markdown, AsciiDoc предлагает гораздо более широкие возможности: атрибуты, перекрёстные ссылки, включение файлов, сложные таблицы и многое другое. При этом он остаётся текстовым форматом, удобным для хранения в Git и совместной работы.

== Заголовки

Заголовки создаются с помощью знаков равенства `=`. Количество знаков соответствует уровню заголовка.

Важно: в документе может быть **только один** заголовок первого уровня, и он должен располагаться в самом начале файла.

[source, asciidoc]
----
= Заголовок первого уровня (название документа)

== Заголовок второго уровня (основные разделы)

=== Заголовок третьего уровня (подразделы)

==== Заголовок четвёртого уровня

===== Заголовок пятого уровня

====== Заголовок шестого уровня
----

Текст, идущий сразу после заголовка первого уровня, называется "введением". В данном документе введение — это абзац после первого заголовка до `toc::[]`.

== Параграфы и разрывы строк

Параграфы разделяются одной или несколькими пустыми строками. Обычный разрыв строки внутри параграфа игнорируется — это позволяет писать исходный текст удобно, не заботясь о длине строк.

[source, asciidoc]
----
Это первый параграф. Он может состоять из нескольких строк в исходном коде,
но при отображении будет единым блоком.

Это второй параграф. Его отделяет пустая строка.
----

Если вам нужно сохранить разрывы строк, используется опция `[%hardbreaks]` перед параграфом:

[source, asciidoc]
----
[%hardbreaks]
Эта строка будет перенесена.
И эта тоже.
А это уже третья строка.
----

Результат:
[%hardbreaks]
Эта строка будет перенесена.
И эта тоже.
А это уже третья строка.

== Выделение текста

Форматирование символов очень похоже на Markdown:

* *Полужирный* — обрамляется звёздочками `*`.

Пример:  *важно*.

* _Курсив_ — обрамляется нижними подчёркиваниями `_`.

Пример: _важно_.

* `Моноширинный` — обрамляется обратными кавычками `` ` ``. Используется для имён переменных, функций, кода в строке. Пример:

Пример: `переменная`.

[source, asciidoc]
----
Все вместе: обратите _внимание_ на *важный* файл `config.json`.
----

Все вместе: Обратите _внимание_ на *важный* файл `config.json`.

TIP: AsciiDoc автоматически преобразует сочетания `` "` "` и `'` `'` в типографские «ёлочки» или “лапки”.

== Списки

AsciiDoc поддерживает маркированные, нумерованные и многоуровневые списки, а также списки описаний.

=== Маркированный список

Используетсят символ `*`. Уровень вложенности задаётся количеством звёздочек.

[source, asciidoc]
----
* Пункт первого уровня
** Подпункт второго уровня
*** Подпункт третьего уровня
* Возврат на первый уровень
----

Результат:

* Пункт первого уровня
** Подпункт второго уровня
*** Подпункт третьего уровня
* Возврат на первый уровень

=== Нумерованный список

Используйте точку `.`. Не нужно следить за нумерацией — можно везде ставить `.`, и AsciiDoc сам проставит правильные цифры.

[source, asciidoc]
----
. Первый шаг
. Второй шаг
.. Подшаг 2.1
.. Подшаг 2.2
... Подшаг 2.2.1
. Третий шаг
----

Результат:

. Первый шаг
. Второй шаг
.. Подшаг 2.1
.. Подшаг 2.2
... Подшаг 2.2.1
. Третий шаг

=== Список описаний (термин-определение)

Создаётся с помощью двойного двоеточия `::`. Удобно для глоссариев или описания параметров.

[source, asciidoc]
----
термин 1:: описание термина 1.
термин 2:: описание термина 2.
----

Результат:

термин 1:: описание термина 1.
термин 2:: описание термина 2.

== Ссылки

=== Внешние ссылки

Оформляются двумя способами: автоматическое преобразование URL или через ключевое слово link (в случае, если файл находится внутри репозитория).

Автоматическоп преобразование URL:
[source, asciidoc]
----
https://systems.education/[Школа Бескова и Богачёвой]
----

Результат: https://systems.education/[Школа Бескова и Богачёвой]

С использованием link:
[source, asciidoc]
----
link: link:/MarkDown.md[Инструкция по Markdown].
----

Резуьтат: link:/MarkDown.md[Инструкция по Markdown].

=== Перекрёстные ссылки внутри документа

Для создания ссылки на другой раздел нужно сначала установить якорь (метку) с помощью
[source, asciidoc]
----
`[[идентификатор]]`,
----
а затем сослаться на него через

[source, asciidoc]
----
`<<идентификатор>>`.
----

Например:
[source, asciidoc]
----
[[Заголовки]]
== Заголовоки
... текст раздела ...

Для более подробной информации смотрите <<Заголовки>>.
----

Ссылка на этот раздел: <<Заголовки>>.

== Изображения

Для вставки изображений используются макросы `image:` (строчное) и `image::` (блочное). Можно задавать размеры и атрибуты.

[source, asciidoc]
----
// Блочное изображение
image::diagram.png[]

// С указанием ширины и альтернативным текстом
image::screenshot.png[Скриншот главного экрана, 800]

// Изображение, которое является ссылкой
image:icon.png[link="https://example.com"]
----

Например:

image::image/example_image_2.png[]

== Таблицы

Таблицы в AsciiDoc — одна из самых мощных функций. Они задаются символами `|===`.

=== Простая таблица

[source, asciidoc]
----
|===
| Имя поля | Тип данных | Обязательное

| user_id
| integer
| да

| username
| string
| да

| bio
| text
| нет
|===
----

Результат:

|===
| Имя поля | Тип данных | Обязательное

| user_id
| integer
| да

| username
| string
| да

| bio
| text
| нет
|===

=== Таблица с настройками

Можно задавать ширину столбцов, выравнивание, заголовок.

[source, asciidoc]
----
[cols="2,3,1", options="header"]
|===
| Имя поля | Тип данных | Обязательное
| user_id  | integer    | да
| username | string     | да
| bio      | text       | нет
|===
----

Результат:

[cols="2,3,1", options="header"]
|===
| Имя поля | Тип данных | Обязательное
| user_id  | integer    | да
| username | string     | да
| bio      | text       | нет
|===

=== Объединение ячеек

Используйте `2+` для объединения по горизонтали и `.2+` для объединения по вертикали.

[source, asciidoc]
----
|===
| Объединение ячеек | Пример

| Обычная ячейка
| Обычная ячейка

| 2+| Две объединённые ячейки по горизонтали

| .2+| Две объединённые ячейки по вертикали
| Первая
| Вторая
|===
----

Результат:
|===
| Объединение ячеек | Пример

| Обычная ячейка
| Обычная ячейка

| 2+| Две объединённые ячейки по горизонтали

| .2+| Две объединённые ячейки по вертикали
| Первая
| Вторая
|===

== Блоки кода

=== Простой блок кода

Для вставки примеров кода можно использовать блок `[source, язык]`.

[source, asciidoc]
----
[source,json]
----
{
"order_id": 12345,
"amount": 1500.50,
"status": "paid"
}
----
----
Результат:

[source,json]
----
{
"order_id": 12345,
"amount": 1500.50,
"status": "paid"
}
----

=== Блок кода с номерами строк и выносками (callouts)

Выноски позволяют объяснить отдельные части кода.

[source, asciidoc]
----
[source,python,linenums]
----
def hello():      # <1>
print("Hello, World!") # <2>
----
<1> Определение функции.
<2> Вывод сообщения в консоль.
----

Результат:

[source,python,linenums]
----
def hello():      # <1>
    print("Hello, World!") # <2>
----
<1> Определение функции.
<2> Вывод сообщения в консоль.

== Предупреждения (Admonitions)

Специальные блоки для привлечения внимания: NOTE, TIP, IMPORTANT, CAUTION, WARNING.

[source, asciidoc]
----
NOTE: Это однострочное примечание.

[WARNING]
====
Будьте осторожны! Эта операция необратима.
Она может привести к потере данных.
====
----

Результат:

NOTE: Это однострочное примечание.

[WARNING]
====
Будьте осторожны! Эта операция необратима.
Она может привести к потере данных.
====

== Атрибуты документа

Атрибуты — это переменные, которые можно определить в начале документа и использовать по всему тексту. Это очень удобно для указания версии, автора или статуса.

В начале этого файла определены атрибуты:

[source, asciidoc]
----
:author: Команда аналитиков
:revnumber: 1.0
:revdate: 2024-03-08
----

Используются они так:

[source, asciidoc]
----
Документ версии {revnumber} от {revdate}.
Автор: {author}.
----

Результат: Документ версии {revnumber} от {revdate}. Автор: {author}.

Атрибуты можно переопределять по ходу документа или задавать при сборке.

== Включение файлов

Большие документы можно разбивать на части и собирать из них итоговый файл с помощью директивы `include::`.

[source, asciidoc]
----
= Руководство пользователя

include::chapter1-intro.adoc[]
include::chapter2-installation.adoc[leveloffset=+1]
include::chapter3-configuration.adoc[]
----

Встраивание контента в AsciiDoc осуществляется через директиву include::, которая позволяет вставить содержимое одного файла в другой при генерации документа.

Например:
[source, asciidoc]
----
include::path/to/file.adoc[]
----

Результат:
include::for_include/include.adoc[]

== Спойлеры

В AsciiDoc для создания сворачиваемых блоков (спойлеров) используется специальный синтаксис collapsible (сворачиваемый) для конвертации.

.Нажмите, чтобы развернуть спойлер
[%collapsible]
====
Здесь находится скрытый текст или содержимое.
Оно может содержать **разметку**, списки и изображения.
====
